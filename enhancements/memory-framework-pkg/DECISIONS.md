# DECISIONS.md
# Permanent decision log. NEVER truncate, summarise, or delete entries.
# Append only. Every architectural or significant decision lives here with its reason.
# If a decision is later reversed, mark it [REVERSED] and add a new entry explaining why.
# Claude reads this at session start. It prevents re-litigating settled decisions.

---

## How to Read This File

Each entry answers three questions:
1. **What** was decided
2. **Why** — the reason that was compelling at the time
3. **Constraints** — what would need to change for this decision to be revisited

Entries are never removed. If circumstances change, add a new entry that references the old one.
